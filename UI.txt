四、交互与逻辑规范
4.1 模式切换逻辑
标定模式	标定类型	显示的输入卡片	隐藏的输入卡片
眼在手外	标记物标定	相机目标点坐标、机器人拍照位姿	机器人目标点坐标
眼在手外	戳点标定	相机目标点坐标、机器人目标点坐标	机器人拍照位姿
眼在手上	标记物标定	相机目标点坐标、机器人拍照位姿	机器人目标点坐标
眼在手上	戳点标定	相机目标点坐标、机器人拍照位姿、机器人目标点坐标	无
4.2 快捷键支持
快捷键	功能
Ctrl + S	保存当前数据
Ctrl + R	重拍
Ctrl + Z	撤销上一次保存
Ctrl + C	复制当前激活输入框的内容
F5	刷新相机图像
F11	全屏切换
4.3 数据验证与反馈
输入框自动验证坐标格式（支持 x,y,z 或 x y z 格式）
格式错误时，输入框边框变红，下方显示 "请输入正确的坐标格式"
所有操作完成后显示 3 秒 Toast 提示（右上角）
数据保存成功：绿色 Toast，文字 "第 X 组数据保存成功"
数据保存失败：红色 Toast，文字 "数据保存失败，请检查输入"
4.4 自动保存机制
每采集一组数据自动备份到本地 backup/calibration_data_YYYYMMDD_HHMMSS.json
软件启动时自动检测是否有未完成的标定任务，提示用户是否恢复
备份文件保留最近 10 次的记录
五、开发注意事项
性能优化：2D/3D 可视化窗口应使用硬件加速，避免卡顿
响应式布局：在 1600×900 分辨率下，右侧辅助区宽度调整为 360px，可视化窗口宽度调整为 580px
多语言支持：所有文字内容应使用资源文件管理，方便后续国际化
错误处理：添加全局异常捕获，避免软件崩溃，崩溃时自动保存当前数据
日志系统：所有操作和错误都应写入本地日志文件，方便排查问题
