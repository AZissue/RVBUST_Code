# 技术支持工作记录管理系统 (Tech Support CRM)

## 项目概述
为3D相机技术支持团队（5人）开发一个内部工作记录管理软件，替代现有的企业微信表格记录方式。核心解决痛点：记录繁琐、客户信息分散、问题跟进困难。

## 技术栈
- 前端：Vue 3 + Element Plus + Vite
- 后端：Python + FastAPI + SQLAlchemy
- 数据库：SQLite（单文件，零配置）
- 部署：现在本地部署，后期可上传到公网

## 核心功能模块

### 1. 首页工作台（/dashboard）
- 顶部统计卡片：待处理数量、处理中数量、等待反馈数量
- 快速记录工单区域：
  - 客户名称输入框（支持模糊搜索现有客户，支持回车新建客户）
  - 问题类型下拉：硬件故障/软件使用/咨询/投诉/其他
  - 问题标题（一句话概括）
  - 详细描述（多行文本）
  - 负责人下拉（预设5个技术员姓名）
  - 优先级：紧急/高/普通/低
  - 保存按钮（保存后清空表单，可连续录入）
- 我的今日待办列表：按紧急程度排序，显示超时预警（红色）

### 2. 客户管理（/customers）
- 客户列表页：表格展示，支持搜索、分页
- 客户字段：名称、联系人、电话、邮箱、地区、行业、创建时间、活跃度标签（高/中/低）
- 客户详情页（核心）：
  - 顶部客户信息卡片
  - 设备清单（关联显示）
  - 互动时间轴：按时间倒序显示所有工单和跟进记录，带时间线样式
  - 快捷操作：[+ 新工单] [编辑客户] [删除]
- 新建/编辑客户弹窗表单

### 3. 工单管理（/tickets）
- 双视图切换：看板视图（Kanban）/ 列表视图
- 看板视图：四列（待处理/处理中/等待反馈/已解决），卡片可拖拽改状态
- 列表视图：表格，支持筛选（按状态/负责人/客户/日期）、排序
- 工单字段：编号、客户（关联）、标题、类型、状态、负责人、优先级、创建时间、解决时间
- 工单详情页：
  - 基本信息展示
  - 状态流转按钮：[开始处理] [等待客户反馈] [标记已解决] [关闭工单]
  - 跟进记录时间轴（可添加备注）
  - 关联客户快捷跳转

### 4. 数据统计（简单版）
- 个人工作台统计：本周处理量、平均响应时长、待办完成率
- 团队看板（可选）：各负责人当前负载

### 5. 自动提醒（后端定时任务）
- 每天9:00和17:00执行
- 检查规则：
  - 待处理状态超过24小时 → 标红提醒
  - 处理中状态超过48小时 → 标红提醒
  - 等待反馈状态超过72小时 → 建议催办
- 提醒方式：页面内消息中心（前端轮询），暂不需要邮件

## 数据库设计（SQLite）

表1：customers（客户档案）
- id INTEGER PRIMARY KEY
- name VARCHAR(100) NOT NULL（唯一）
- contact_person VARCHAR(50)
- phone VARCHAR(20)
- email VARCHAR(100)
- region VARCHAR(50)
- industry VARCHAR(50)
- created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
- last_contact_at TIMESTAMP
- activity_level VARCHAR(20) DEFAULT 'medium' -- high/medium/low

表2：devices（设备信息，可选关联）
- id INTEGER PRIMARY KEY
- customer_id INTEGER FOREIGN KEY
- model VARCHAR(50)
- serial_number VARCHAR(100)
- firmware_version VARCHAR(50)
- purchase_date DATE
- created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

表3：tickets（工单）
- id INTEGER PRIMARY KEY
- customer_id INTEGER FOREIGN KEY NOT NULL
- device_id INTEGER FOREIGN KEY（可为空）
- title VARCHAR(200) NOT NULL
- category VARCHAR(50) -- 硬件故障/软件使用/咨询/投诉/其他
- description TEXT
- solution TEXT（解决后填写）
- status VARCHAR(20) DEFAULT 'pending' -- pending/processing/waiting_feedback/resolved/closed
- priority VARCHAR(20) DEFAULT 'normal' -- urgent/high/normal/low
- assignee VARCHAR(50)
- created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
- started_at TIMESTAMP（开始处理时间）
- resolved_at TIMESTAMP（解决时间）
- closed_at TIMESTAMP（关闭时间）

表4：follow_ups（跟进记录）
- id INTEGER PRIMARY KEY
- ticket_id INTEGER FOREIGN KEY NOT NULL
- content TEXT NOT NULL
- type VARCHAR(20) DEFAULT 'comment' -- comment/status_change/system
- created_by VARCHAR(50)
- created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

## API设计规范

RESTful API，统一返回格式：
{
  "code": 200,
  "message": "success",
  "data": {}
}

核心接口：
- GET    /api/dashboard/stats          -- 首页统计数据
- GET    /api/dashboard/todos          -- 我的待办列表
- POST   /api/tickets/quick            -- 快速创建工单（首页用）
- GET    /api/tickets                  -- 工单列表（支持筛选分页）
- GET    /api/tickets/{id}             -- 工单详情
- POST   /api/tickets                  -- 创建工单
- PUT    /api/tickets/{id}             -- 更新工单
- PUT    /api/tickets/{id}/status      -- 更新状态（专用接口，记录时间）
- GET    /api/customers                -- 客户列表（支持搜索）
- GET    /api/customers/{id}           -- 客户详情（包含时间轴数据）
- POST   /api/customers                -- 创建客户
- PUT    /api/customers/{id}           -- 更新客户
- DELETE /api/customers/{id}           -- 删除客户
- GET    /api/customers/search?q=xxx   -- 客户搜索（自动补全用）
- POST   /api/follow_ups               -- 添加工单跟进记录

## 前端路由
- /           -- 工作台（首页）
- /customers  -- 客户列表
- /customers/:id -- 客户详情
- /tickets    -- 工单管理
- /tickets/:id   -- 工单详情

## 关键交互细节

1. 客户名称自动补全：
   - 输入时实时搜索（防抖300ms）
   - 显示匹配列表：名称 + 联系人 + 地区
   - 选中后自动填充客户信息
   - 无匹配时显示"按回车创建新客户：[输入内容]"

2. 看板拖拽：
   - 卡片可跨列拖拽
   - 拖拽后调用状态更新接口
   - 状态变更自动记录到follow_ups（系统类型）

3. 时间轴展示：
   - 左侧时间线，右侧内容卡片
   - 不同类型不同颜色：工单创建(蓝)、状态变更(橙)、跟进备注(灰)
   - 时间显示相对时间（2小时前、昨天、2026-04-20）

4. 表单优化：
   - 保存成功后保持表单打开，清空内容，聚焦到客户输入框（连续录入场景）
   - 提供"保存并关闭"和"保存并继续"两个按钮

## 开发顺序（按此顺序生成代码）

请按以下顺序实现，每完成一个模块可独立测试：

1. 后端基础：数据库模型 + API框架 + 示例数据
2. 前端基础：Vue项目搭建 + 路由 + API封装
3. 首页工作台：快速记录表单 + 统计卡片 + 待办列表
4. 客户管理：列表 + 搜索 + 详情页（含时间轴）
5. 工单管理：列表视图 + 看板视图 + 详情页 + 状态流转
6. 提醒功能：后端定时任务 + 前端消息通知
7. 部署配置：前后端整合，单端口运行

## 注意事项

- 不要过度设计，先跑通MVP
- 前端UI使用Element Plus默认主题，无需自定义样式
- 错误处理：API返回统一错误格式，前端用ElMessage提示
- 图片/附件功能可暂不支持，如需支持用本地存储路径
- 无需用户登录系统，内部工具信任使用
- 代码注释使用中文，方便团队维护
